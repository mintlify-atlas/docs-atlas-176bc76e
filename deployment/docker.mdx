Docker provides a containerized way to run MCP Connector with all dependencies included, ensuring consistency across different environments.

## Prerequisites

Before deploying with Docker, ensure you have:

- Docker installed on your machine ([download here](https://docs.docker.com/get-docker/))
- Your TypingMind authentication token
- Basic familiarity with Docker commands

## Building the Docker image

The MCP Connector includes a Dockerfile based on Node.js 23 with Python and uv support for running various MCP servers.

<Steps>
  <Step title="Clone or download the repository">
    Get the MCP Connector source code with the Dockerfile:
    
    ```bash
    git clone https://github.com/TypingMind/typingmind-mcp.git
    cd typingmind-mcp
    ```
  </Step>
  
  <Step title="Build the Docker image">
    Navigate to the directory containing the Dockerfile and build the image:
    
    ```bash
    docker build -t mcp-connector .
    ```
    
    This creates an image named `mcp-connector`. You can use any name you prefer.
  </Step>
  
  <Step title="Verify the build">
    Check that the image was created successfully:
    
    ```bash
    docker images | grep mcp-connector
    ```
    
    Expected output:
    ```
    mcp-connector    latest    abc123def456    2 minutes ago    850MB
    ```
  </Step>
</Steps>

## Dockerfile overview

The MCP Connector Dockerfile includes:

```dockerfile
# Use an official Node.js runtime as a parent image (slim variant for better security)
FROM node:23-slim

# Install Python, pip, and other dependencies
RUN apt-get update && \
    apt-get install -y python3 python3-pip && \
    rm -rf /var/lib/apt/lists/*

# Install uv via pip
RUN pip3 install uv --break-system-packages 

# Set the working directory in the container
WORKDIR /app

# Install pnpm globally
RUN npm install -g pnpm

# Copy package.json and pnpm-lock.yaml first to leverage Docker cache
COPY package.json pnpm-lock.yaml ./

# Install only production dependencies
RUN pnpm install --prod

# Copy the rest of the application source code
COPY . .

# Set the default port the app will run on
ENV PORT=50880

# Make port 50880 available to the world outside this container
EXPOSE 50880

# Define the command to run the app
CMD ["node", "bin/index.js"]
```

<Note>
  The image includes Python 3 and uv to support MCP servers that require Python dependencies.
</Note>

## Running the container

### Basic HTTP deployment

Run the container in detached mode with port mapping:

```bash
docker run -d -p 50880:50880 --name mcp-connector-instance mcp-connector <auth-token>
```

- `-d`: Run in detached mode (background)
- `-p 50880:50880`: Map port 50880 from container to host
- `--name mcp-connector-instance`: Name the container for easy reference
- `<auth-token>`: Your TypingMind authentication token

### Using environment variables

Pass the authentication token via environment variable:

```bash
docker run -d \
  -p 50880:50880 \
  -e MCP_AUTH_TOKEN=<auth-token> \
  --name mcp-connector-instance \
  mcp-connector
```

### Using a custom port

To run on a different port, map the host port to a custom container port:

```bash
docker run -d \
  -p 8080:12345 \
  -e PORT=12345 \
  --name mcp-connector-instance \
  mcp-connector <auth-token>
```

This maps port 8080 on your host to port 12345 inside the container.

<Note>
  Remember to update your TypingMind configuration to use the correct port (e.g., `http://localhost:8080`).
</Note>

## Running with HTTPS

For secure connections, mount SSL certificate files and configure HTTPS:

<Steps>
  <Step title="Prepare SSL certificates">
    Ensure you have:
    - SSL certificate file (e.g., `certificate.crt`)
    - Private key file (e.g., `privatekey.key`)
  </Step>
  
  <Step title="Run with volume mounts">
    Mount the certificate files into the container:
    
    ```bash
    docker run -d \
      -p 50880:50880 \
      -e PORT=50880 \
      -e CERTFILE=/certs/certificate.crt \
      -e KEYFILE=/certs/privatekey.key \
      -v /path/to/your/certificate.crt:/certs/certificate.crt:ro \
      -v /path/to/your/privatekey.key:/certs/privatekey.key:ro \
      --name mcp-connector-instance \
      mcp-connector <auth-token>
    ```
    
    Replace `/path/to/your/` with the actual paths on your host machine.
  </Step>
</Steps>

<Warning>
  The `:ro` flag mounts the certificate files as read-only for security. Never commit certificates to your Docker image.
</Warning>

## Container management

### Viewing logs

View real-time logs from the running container:

```bash
docker logs mcp-connector-instance
```

Follow logs continuously:

```bash
docker logs -f mcp-connector-instance
```

View the last 100 lines:

```bash
docker logs --tail 100 mcp-connector-instance
```

### Checking container status

List running containers:

```bash
docker ps
```

Expected output:
```text
CONTAINER ID   IMAGE           COMMAND                  CREATED          STATUS          PORTS                      NAMES
abc123def456   mcp-connector   "node bin/index.js"      5 minutes ago    Up 5 minutes    0.0.0.0:50880->50880/tcp   mcp-connector-instance
```

### Stopping the container

Stop the running container:

```bash
docker stop mcp-connector-instance
```

This gracefully stops the container, allowing it to clean up resources.

### Starting a stopped container

Restart a stopped container:

```bash
docker start mcp-connector-instance
```

### Removing the container

Remove a stopped container:

```bash
docker rm mcp-connector-instance
```

Force remove a running container:

```bash
docker rm -f mcp-connector-instance
```

<Warning>
  Removing a container deletes all data inside it. Configuration and logs will be lost unless persisted with volumes.
</Warning>

## Advanced configuration

### Persisting data with volumes

To persist configuration or data, create a volume:

```bash
docker volume create mcp-connector-data

docker run -d \
  -p 50880:50880 \
  -e MCP_AUTH_TOKEN=<auth-token> \
  -v mcp-connector-data:/app/data \
  --name mcp-connector-instance \
  mcp-connector
```

### Setting custom timeout

Configure tool call timeout (in milliseconds):

```bash
docker run -d \
  -p 50880:50880 \
  -e MCP_AUTH_TOKEN=<auth-token> \
  -e TIMEOUT=120000 \
  --name mcp-connector-instance \
  mcp-connector
```

### Auto-restart on failure

Configure the container to restart automatically:

```bash
docker run -d \
  --restart unless-stopped \
  -p 50880:50880 \
  -e MCP_AUTH_TOKEN=<auth-token> \
  --name mcp-connector-instance \
  mcp-connector
```

Restart policies:
- `no`: Never restart (default)
- `always`: Always restart the container
- `unless-stopped`: Restart unless manually stopped
- `on-failure`: Restart only on failure

### Resource limits

Limit CPU and memory usage:

```bash
docker run -d \
  -p 50880:50880 \
  -e MCP_AUTH_TOKEN=<auth-token> \
  --memory="512m" \
  --cpus="1.0" \
  --name mcp-connector-instance \
  mcp-connector
```

## Docker Compose

For easier management, use Docker Compose:

<Steps>
  <Step title="Create a docker-compose.yml file">
    ```yaml docker-compose.yml
    version: '3.8'
    
    services:
      mcp-connector:
        build: .
        container_name: mcp-connector-instance
        ports:
          - "50880:50880"
        environment:
          - MCP_AUTH_TOKEN=<auth-token>
          - TIMEOUT=60000
        restart: unless-stopped
        volumes:
          # Uncomment for HTTPS
          # - /path/to/certificate.crt:/certs/certificate.crt:ro
          # - /path/to/privatekey.key:/certs/privatekey.key:ro
        # Uncomment for HTTPS
        # environment:
        #   - CERTFILE=/certs/certificate.crt
        #   - KEYFILE=/certs/privatekey.key
    ```
  </Step>
  
  <Step title="Start with Docker Compose">
    ```bash
    docker-compose up -d
    ```
  </Step>
  
  <Step title="Manage with Docker Compose">
    <CodeGroup>
    ```bash View logs
    docker-compose logs -f
    ```
    
    ```bash Stop services
    docker-compose down
    ```
    
    ```bash Restart services
    docker-compose restart
    ```
    
    ```bash Rebuild and restart
    docker-compose up -d --build
    ```
    </CodeGroup>
  </Step>
</Steps>

## Networking

### Accessing from host machine

When running locally, access the container at:

```text
http://localhost:50880
```

### Accessing from other machines

If running on a remote server, use the server's IP address:

```text
http://your-server-ip:50880
```

Ensure firewall rules allow traffic on port 50880.

### Custom Docker network

Create a custom network for better isolation:

```bash
docker network create mcp-network

docker run -d \
  --network mcp-network \
  -p 50880:50880 \
  -e MCP_AUTH_TOKEN=<auth-token> \
  --name mcp-connector-instance \
  mcp-connector
```

## Troubleshooting

### Container fails to start

Check container logs for errors:

```bash
docker logs mcp-connector-instance
```

Common issues:
- Port already in use: Change the host port mapping
- Invalid auth token: Verify the token is correct
- Missing environment variables: Check `-e` flags

### Port conflicts

If port 50880 is already in use:

```bash
# Use a different host port
docker run -d -p 8080:50880 -e MCP_AUTH_TOKEN=<auth-token> --name mcp-connector-instance mcp-connector
```

### Certificate permission errors

Ensure certificate files are readable:

```bash
chmod 644 /path/to/certificate.crt
chmod 600 /path/to/privatekey.key
```

### Container exits immediately

Verify the authentication token is provided:

```bash
# This will fail
docker run -d -p 50880:50880 --name mcp-connector-instance mcp-connector

# Correct usage
docker run -d -p 50880:50880 --name mcp-connector-instance mcp-connector <auth-token>
```

### Inspecting container details

View detailed container information:

```bash
docker inspect mcp-connector-instance
```

### Accessing container shell

For debugging, access the container shell:

```bash
docker exec -it mcp-connector-instance /bin/bash
```

## Production best practices

<Steps>
  <Step title="Use specific image tags">
    Instead of `latest`, tag your images with version numbers:
    
    ```bash
    docker build -t mcp-connector:1.3.1 .
    docker run -d -p 50880:50880 --name mcp-connector-instance mcp-connector:1.3.1 <auth-token>
    ```
  </Step>
  
  <Step title="Enable auto-restart">
    Always use a restart policy for production:
    
    ```bash
    docker run -d --restart unless-stopped -p 50880:50880 -e MCP_AUTH_TOKEN=<auth-token> --name mcp-connector-instance mcp-connector
    ```
  </Step>
  
  <Step title="Use environment files">
    Store sensitive data in environment files:
    
    ```bash .env
    MCP_AUTH_TOKEN=your-secret-token
    TIMEOUT=120000
    PORT=50880
    ```
    
    ```bash
    docker run -d -p 50880:50880 --env-file .env --name mcp-connector-instance mcp-connector
    ```
  </Step>
  
  <Step title="Monitor resource usage">
    Regularly check container resource consumption:
    
    ```bash
    docker stats mcp-connector-instance
    ```
  </Step>
  
  <Step title="Implement health checks">
    Add health checks to your Docker Compose configuration:
    
    ```yaml
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:50880/ping"]
      interval: 30s
      timeout: 10s
      retries: 3
    ```
  </Step>
</Steps>

## Next steps

After deploying with Docker:

1. Note your server address (e.g., `http://localhost:50880` or `http://your-server-ip:50880`)
2. Configure the address in TypingMind at [www.typingmind.com/mcp](https://www.typingmind.com/mcp)
3. Verify connectivity using the `/ping` endpoint
4. Start using MCP tools in TypingMind

<Note>
  For production deployments, consider using orchestration tools like Kubernetes or Docker Swarm for advanced scaling and management capabilities.
</Note>
