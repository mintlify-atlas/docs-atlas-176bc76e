## Prerequisites

Before you begin, ensure you have:

- **Node.js 14.0.0 or later** installed on your system
- **Auth token** from TypingMind (get it from [typingmind.com/mcp](https://www.typingmind.com/mcp))
- **Port availability**: Ports 50880 or 50881 should be available (or specify a custom port)

<Note>
You don't need to install anything globally. MCP Connector runs directly with `npx`.
</Note>

## Installation and setup

<Steps>
  <Step title="Get your auth token">
    Visit [typingmind.com/mcp](https://www.typingmind.com/mcp) and copy your authentication token. You'll need this to start MCP Connector.

    <Warning>
    Keep your auth token secure. Anyone with this token can access your MCP Connector instance.
    </Warning>
  </Step>

  <Step title="Start MCP Connector">
    Run MCP Connector using `npx`. Replace `your-auth-token` with your actual token:

    <CodeGroup>
    ```bash Command line argument
    npx @typingmind/mcp@latest your-auth-token
    ```

    ```bash Environment variable
    MCP_AUTH_TOKEN=your-auth-token npx @typingmind/mcp@latest
    ```

    ```bash Custom port
    PORT=8080 npx @typingmind/mcp@latest your-auth-token
    ```
    </CodeGroup>

    You should see output similar to:

    ```bash
    ✓ MCP runner server running on http://0.0.0.0:50880
    Note: You must keep the server running in the background in order to use MCP in TypingMind.
    ```
  </Step>

  <Step title="Verify the server is running">
    Test the health check endpoint to confirm MCP Connector is running:

    ```bash
    curl -X GET http://localhost:50880/ping \
      -H "Authorization: Bearer your-auth-token"
    ```

    You should receive:

    ```json
    {
      "status": "ok"
    }
    ```

    <Note>
    If you specified a custom port, replace `50880` with your port number.
    </Note>
  </Step>

  <Step title="Configure MCP servers">
    Start one or more MCP servers by sending a POST request to the `/start` endpoint. Here's an example configuration:

    ```bash
    curl -X POST http://localhost:50880/start \
      -H "Authorization: Bearer your-auth-token" \
      -H "Content-Type: application/json" \
      -d '{
        "mcpServers": {
          "filesystem": {
            "command": "npx",
            "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"],
            "env": {}
          }
        }
      }'
    ```

    You should receive a success response:

    ```json
    {
      "message": "All MCP clients started successfully",
      "clients": [
        {
          "id": "filesystem",
          "message": "MCP client started successfully"
        }
      ]
    }
    ```
  </Step>

  <Step title="Connect to TypingMind">
    In TypingMind:

    1. Navigate to the MCP integration page at [typingmind.com/mcp](https://www.typingmind.com/mcp)
    2. Enter your MCP Connector server address:
       - Local: `http://localhost:50880`
       - Remote: `http://your-server-ip:50880`
    3. Enter your authentication token
    4. Save the configuration

    Your MCP servers are now available in TypingMind!
  </Step>
</Steps>

## Verify everything works

List all active MCP clients and their available tools:

```bash
curl -X GET http://localhost:50880/clients \
  -H "Authorization: Bearer your-auth-token"
```

You should see a list of your running MCP servers with their tools:

```json
[
  {
    "id": "filesystem",
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"],
    "createdAt": "2026-03-03T10:30:00.000Z",
    "tools": ["read_file", "write_file", "list_directory"]
  }
]
```

## Common configuration options

### Custom timeout for tool calls

Set a custom timeout (in milliseconds) for tool execution:

```bash
# Set 2 minute timeout
TIMEOUT=120000 npx @typingmind/mcp@latest your-auth-token
```

The default timeout is 60 seconds (60000ms).

### Custom hostname

Bind to a specific hostname instead of the default `0.0.0.0`:

```bash
HOSTNAME=localhost npx @typingmind/mcp@latest your-auth-token
```

### HTTPS support

Enable HTTPS with SSL certificates:

```bash
CERTFILE=./certificate.crt KEYFILE=./privatekey.key \
  npx @typingmind/mcp@latest your-auth-token
```

## Keep the server running

<Warning>
MCP Connector must remain running for TypingMind to access your MCP servers. If you close the terminal or stop the process, TypingMind will lose connection.
</Warning>

For persistent operation, consider:

- **Process managers**: Use `pm2` to keep MCP Connector running in the background
- **Docker**: Run MCP Connector in a Docker container
- **System services**: Set up as a systemd service on Linux

See the [Deployment guide](/deployment/local) for detailed instructions.
